PROJECT OVERVIEW
================================================================================
Flow-Factory is a comprehensive, distributed training framework for multi-modal
generative models using Flow Matching / Diffusion techniques. The codebase
demonstrates professional software architecture with:
- 27,091 lines of Python code across 63 files
- 4 training algorithms (DPO, GRPO, NFT, AWM)
- 11 supported models (image & video generation)
- 5+ reward model implementations
- Multi-GPU distributed training (Accelerate + DeepSpeed)
- Multi-modal support (text, images, videos)
- Extensible plugin architecture

TENSOR FORMAT CONVENTIONS
================================================================================
Image:
  Single: (C, H, W)              ← C=3 (RGB), H/W = 512, 1024, etc.
  Batch: (B, C, H, W)             ← B = batch size
  Values: [0, 255] or [-1, 1]     ← Auto-detected and normalized

Video:
  Single: (T, C, H, W)            ← T = temporal frames
  Batch: (B, T, C, H, W)          ← B = batch size
  Values: [0, 255] or [-1, 1]     ← Auto-detected and normalized

NumPy (internal):
  Image: (H, W, C)                ← Different order!
  Video: (T, H, W, C)             ← Converted on load/save

Auto-standardization provided by:
  - standardize_image_batch() in utils/image.py
  - standardize_video_batch() in utils/video.py


TRAINING PIPELINE FLOW
================================================================================
Input YAML Config
    ↓
Arguments.load_from_yaml()
    ↓
┌───────────────────────────────────────────────────┐
│ 1. DATA LOADING & PREPROCESSING (Epoch Start)    │
│   - Load JSONL/TXT file                          │
│   - Tokenize text → prompt_ids, prompt_embeds    │
│   - Load images → VAE encode → cache             │
│   - Load videos → resample → VAE encode → cache  │
└───────────────────────────────────────────────────┘
    ↓
┌───────────────────────────────────────────────────┐
│ 2. SAMPLE GENERATION (Per Batch)                  │
│   - K-repeat: generate K samples per prompt      │
│   - Model forward: latent + denoise steps        │
│   - Trajectory: collect timesteps, latents       │
│   - Output: T2ISample/T2VSample/etc.             │
└───────────────────────────────────────────────────┘
    ↓
┌───────────────────────────────────────────────────┐
│ 3. REWARD COMPUTATION                            │
│   - CLIP: image-text alignment                   │
│   - PickScore: aesthetic score                   │
│   - OCR: text recognition match                  │
│   - VLM: vision-language evaluation              │
│   - Aggregate: weighted multi-reward score       │
└───────────────────────────────────────────────────┘
    ↓
┌───────────────────────────────────────────────────┐
│ 4. ADVANTAGE COMPUTATION                          │
│   - Group K samples per prompt                   │
│   - Base: min/mean/std of group scores           │
│   - Advantage: (sample_reward - base) / std      │
│   - Normalize per batch                          │
└───────────────────────────────────────────────────┘
    ↓
┌───────────────────────────────────────────────────┐
│ 5. GRADIENT UPDATE                               │
│   - Predict noise: pred = model(latent, t)       │
│   - Target: ground truth noise                   │
│   - MSE loss: (pred - target)^2                  │
│   - Weight: loss * advantage                     │
│   - Backward + optimizer step                    │
│   - Update EMA model (if enabled)                │
└───────────────────────────────────────────────────┘
    ↓
┌───────────────────────────────────────────────────┐
│ 6. LOGGING & CHECKPOINTING                       │
│   - Log metrics (loss, reward, advantage)        │
│   - Save to W&B/TensorBoard/SwanLab             │
│   - Checkpoint model every N steps               │
│   - Save optimizer & scheduler state             │
└───────────────────────────────────────────────────┘
    ↓
[Repeat for next batch or epoch]

COMMON TASKS:

1. Run Training
   $ python -m flow_factory.train config.yaml

2. Load Model
   model = load_model(model_name="flux.1", ...)

3. Create Sample
   sample = T2ISample(prompt="...", image=tensor)

4. Stack Samples
   batch = BaseSample.stack([sample1, sample2])

5. Compute Rewards
   scores = reward_model(prompt=[...], image=[...])

6. Convert Media
   image_tensor = standardize_image_batch(pil_image, 'pt')
   video_tensor = standardize_video_batch(frames, 'pt')

VERIFICATION CHECKLIST FOR EXTENSIONS
================================================================================
When implementing new features, verify:

Data Type Conversions
  ✓ Tensor formats follow conventions (C,H,W) or (T,C,H,W)
  ✓ Value ranges auto-detected [-1,1], [0,1], [0,255]
  ✓ NumPy/PIL/PyTorch conversions work both directions

Sample Types
  ✓ Extends BaseSample or conditioning sample type
  ✓ Auto-standardization in __post_init__
  ✓ Unique ID computation for grouping

Component Integration
  ✓ Extends proper ABC base class
  ✓ Registered in corresponding registry
  ✓ Type hints complete and correct
  ✓ Docstrings document behavior
  ✓ Multi-GPU compatible (if applicable)

Configuration
  ✓ Example config in examples/ directory
  ✓ All required fields in YAML
  ✓ Defaults provided for optional fields

Testing
  ✓ Works with different batch sizes
  ✓ Compatible with distributed training
  ✓ Memory usage reasonable
  ✓ Edge cases handled (empty tensors, None values)
